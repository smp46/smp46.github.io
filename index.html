Undergrad Projects

Maze Game

This is a game I completed in my Intro to Java class. I went far beyond what they asked of us, so here it is.

A goose named Sir Wobbleton is stuck in a dynamically loaded maze (of your choosing), you must get him out (or simply press 'Q' and allow the maze solver to solve the maze). Mazes can be loaded from any file that matches the given format (Theoretically can be infinitely big). The game also can be loaded in three different ways: CLI mode, GUI no-asset mode, and GUI asset mode. Due to licensing, I cannot include the asset folder in the source code, however do not worry I implemented an MD5 checksum check to verify all assets before displaying. If the assets don't match the checksum, the game will work just fine.

UQunscramble

This project was completed in an Intro to C class. As much as I love C, this is a terrible use of the language, nevertheless here it is for your viewing pleasure. Memory leak free.

UQunscramble is a simple word game that can be played in the terminal. You can provide 4 to 13 letters, or let the program generate a random set of letters for you. With these you must create as many words as possible. The game will then check your words against a dictionary and give you a score based on the words you created. A word with as many letters as you have been given gives a bonus 10 points. Try to get the highest score possible!

RightClickVirusTotal

RightClickVirusTotal is a Python program that provides a simple local interface for the VirusTotal API, via the vt-py library. Designed to be added to the Windows context menu, it allows you to easily check any files for viruses before running them.

Key Features

  1. Utilizes the VirusTotal API to upload files and retrieve analysis results.
  2. Displays comprehensive statistics on the file's detection status, including harmless, malicious, suspicious, and undetected classifications.
  3. Built with a user-friendly graphical interface using the Tkinter library.

SmartGarage (Work in Progress)

This is a multi-part project that I am currently working on. The goal is to "smartify" my garage without needing any physical modifications to the garage itself. The first part, is completed and shown on the left. Using PyTorch and a small amount of python, I have trained a Machine Learning image recognition model to detect if my Garage Door is open or closed based on the security camera feed. If the door is open for longer than 10 minutes, I receive an email notification. I also implemented a custom flask API to intergrate with my Docker Homepage, as shown on the left.

The next part of the project is to implement a system that can open and close the garage door remotely and non-intrusively. This is very work in progress, and is the hard part. My current plan is to use a Raspberry Pi and a low cost RF transciver to capture and replay the signal from the garage door remote.
